# Logit Lens 탐구 — LLM 내부를 선형으로 들여다보기

학생 탐구 주제: **"비선형 함수를 선형으로 재정렬하면 분기를 파악할 수 있나?"**

## 실행
```bash
pip install -r requirements_lens.txt
streamlit run logit_lens_app.py
```
처음 실행 시 gpt2 모델(~500MB)을 자동 다운로드. CPU로도 동작(느림).

## 원리
트랜스포머는 층마다 residual stream 벡터를 갱신하고,
**최종** 벡터에 unembedding 행렬을 곱해 다음 토큰을 예측한다.

→ 그럼 **중간 층** 벡터에도 같은 걸 곱해보면?
  그 층에서 모델이 "무슨 토큰을 생각 중이었는지" 읽힌다. 이게 logit lens.

이것이 곧 **선형 재정렬**이다. 비선형(softmax/GELU/LayerNorm)을 거친
중간 상태를, 최종 출력용 선형 사상으로 억지로 읽는 것. 오차가 있고,
**그 오차 자체가 관찰 대상**이다.

## 탭 구성
1. **층별 예측 보기** — 층을 올라가며 예측이 어떻게 변하나
   - LayerNorm 체크박스를 꺼보면 초기 층이 깨지는 걸 볼 수 있음 (스케일 의존 실증)
2. **특정 토큰 추적** — "Paris" 확률이 몇 층에서 솟는가
3. **선형화의 한계** — 왜 '흐름'은 보이는데 '분기'는 안 보이는가

## 탐구해볼 것
- 사실 질문("에펠탑은 어느 도시에")과 문법 질문에서 확정 층이 다른가?
- 답이 확정되는 층이 문장 길이/난이도에 따라 달라지나?
- LayerNorm을 끄면 왜 초기 층이 특히 망가지나?
- 확률이 가장 크게 뛴 층 앞뒤에서 무슨 일이?

## 핵심 결론(미리 스포일러 아님 — 직접 확인할 것)
선형 재정렬로 **기여도 궤적**은 얻지만 **분기 추적**은 얻지 못한다.
트랜스포머 안에 이산적 분기가 애초에 없기 때문 — 있는 건 연속 흐름이다.

### 더 나아가려면
